## 2026-03-13 - [Missing `aria-label`s on icon-only `Button` components]
**Learning:** In this Nuxt/Vue application using Shadcn-ui components, there is a recurring pattern of `Button` components configured with `variant="ghost"` and `size="icon"` that lack `aria-label` attributes. This presents a critical accessibility issue, as screen readers cannot announce the purpose of these buttons. Examples include Search, Wishlist, User Menu, Mobile Menu toggle, Cart, Remove Item, and Quantity increment/decrement buttons.
**Action:** When working with this app's components, particularly `Button` with `size="icon"`, always verify and add descriptive `aria-label` attributes in Turkish to ensure full screen reader accessibility. Also consider using `aria-expanded` for toggle buttons and `aria-live` for dynamic text like cart item quantities.
